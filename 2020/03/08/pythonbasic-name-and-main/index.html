如何理解if __name__ == '__main__':

一句话 Python 概念系列

python程序是自上而下执行的，不需要main入口 。

在python编译器读取源文件的时候会执行它找到的所有代码，而在执行之前会根据当前运行的模块是否为主程序而定义变量name的值为main还是模块名。

因此，该判断语句为真的时候，说明当前运行的脚本为主程序，而非主程序所引用的一个模块。当判断语句为假的时候，说明当前运行的主程序并不是该语句所在的文件，该语句所在的文件是作为一个模块被引用，这时候name 被定义为了该语句所在的模块名字（也就是文件名）。

一个python文件通常有两种使用方法：
第一是作为脚本直接执；
第二是 import 到其他的 python 脚本中被调用（模块重用）执行。

因此if __name__ == '__main__':的作用就是控制这两种情况执行代码的过程，在if __name__ == '__main__':下的代码只有在第一种情况下（即文件作为脚本直接执行）才会被执行，而 import 到其他脚本中是不会被执行的。

if __name__ == '__main__':的运行原理
每个python模块（python文件）都包含内置的变量__name__，当该模块被直接执行时，__name__等于文件名（包含后缀.py）；如果该模块 import 到其他模块中，则该模块的 __name__ 等于模块名称（不包含后缀.py）。而__main__始终指当前执行模块的名称（包含后缀.py）。进而当模块被直接执行时，name == ‘main’ 结果为真。

看个例子：
文件一：

# file demo.py
def func():
    print("func() in demo.py")
print("top-level in demo.py")

if __name__ == "__main__":
    print("demo.py is being run directly")
else:
    print("demo.py is being imported into another module")

文件二：

# file test.py
import one        # start executing demo.py
print("top-level in test.py")
one.func()

if __name__ == "__main__":
    print("test.py is being run directly")
else:
    print("test.py is being imported into another module")

当运行python demo.py，输出:

top-level in demo.py
demo.py is being run directly

当运行python test.py,输出:

top-level in demo.py
demo.py is being imported into another module
top-level in test.py
func() in demo.py
test.py is being run directly

可见，当test.py脚本里导入one模块时，也就是import one时，程序会执行demo.py模块，且对one来说，name变量的值就是one（模块名）了。

小结：
__name__ 是一个系统变量（前后加了双下划线为系统变量，普通变量不能如此命名）

1、如果当前模块为主模块（即调用其他模块的模块），那么此模块名字即为'__main__'；

2、如果当前模块被import，那么此模块名字即为文件名字（不加后边的.py）。

很多朋友对下面这条语句很不理解

if name == ‘main‘:
今天，我就详细的解释这条语句的作用
在同一个文件夹下新建两个脚本，分别命名为demo1.py，demo2.py
叶子分割线
demo1.py

#coding=utf-8

print ‘demo1’,name

def myprint():
print u’我是函数myprint’

myprint()

现在，执行demo1.py脚本，输出的内容为

demo1 main
我是函数myprint
对于上面的代码，要明确以下几点

def myprint() 是函数的定义，函数定义后，自己不会执行，需要调用
myprint()是调用函数的过程，函数里的代码会执行
叶子分割线
模块化
工作中，完成一项工作，不可能只用一个脚本，而是许许多多的脚本，这就是demo2.py存在的意义，在demo2.py里要调用demo1.py里的myprint函数。
此时，你心里难免会产生疑问，为什么不在demo2.py里顶一个myprint函数呢？之所以不这么做，是因为不能把所有的代码都写在一个脚本里，就好比你上学时的笔记按照学科划分是一样的，数学一个笔记，物理一个笔记。
编程里的这些做法并不新奇，那么多函数，完全可以按照功能写在不同的脚本里，处理文件的函数放在一个脚本里，处理日期的函数放在一个脚本里，处理数据库的函数放在一个脚本里。
叶子分割线
demo2.py

#coding=utf-8
from demo1 import myprint

print ‘demo2’,name
myprint()
执行demo2.py脚本，输出结果为

demo1 demo1
我是函数myprint
demo2 main
我是函数myprint
为什么是上面的输出结果呢？
叶子分割线
被引用模块
首先要明确一点，from demo1 import myprint 这条语句执行时，demo1里可以执行的代码都会被执行，这条语句的意思是从demo1 模块里引入 myprint方法
执行demo1.py时，print ‘demo1’,name 的输出结果是’demo1’ main ,而在执行demo2.py时，demo1.py里的 print ‘demo1’,name 也会执行的，但是结果却是 demo1 demo1 ，这是为什么呢？
在python中，当一个脚本被执行时，name 的值是main ，而当这个脚本被引用时，name 的值是脚本的名称
在demo2.py脚本里，我只想执行一次myprint函数，但实际却执行了两次，因为在demo1.py脚本里，有一次myprint函数调用的过程，而前面已经说到，模块被引用时，可执行代码都会自动执行，因此myprint函数被执行了两次，在demo1.py里，你想测试myprint函数，但想不到别的模块引用demo1模块时这条函数调用语句会被执行，这个问题该怎么处理呢？
叶子分割线
程序入口
这就是if name == ‘main‘: 这条语句的作用，这个条件语句判断自己是不是当前被执行的脚本，就是第一个被执行的脚本，相当于c语言里的入口函数,接下来，修改前面的两个脚本
叶子分割线
demo1.py

#coding=utf-8
def myprint():
print u’我是函数myprint’

if name == ‘main‘:
myprint()
叶子分割线
demo2.py

#coding=utf-8
from demo1 import myprint

if name == ‘main‘:
myprint()
这样，执行demo2.py时，会引用demo1.py ，demo1.py里的name 是demo1，不等于main，因此myprint()不会被执行，而demo2.py里的name 等于 main，所以myprint()会被执行
而执行demo1.py时，name 等于 main ，因此myprint函数被调用，通常，这么做是为了在编写脚本时测试函数是否编写的正确，但是这些测试的代码不希望在被引用时执行，那么就必须使用这个方法
